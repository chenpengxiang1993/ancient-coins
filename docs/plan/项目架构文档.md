# 中国古代钱币图鉴 — 项目架构文档

## 项目概览

中国古代钱币图鉴是一个 React 18 单页应用，收录 26 个朝代、443 枚金属铸币的数据与图片。用户可按朝代浏览、搜索钱币，查看铸造工艺、版别体系和估价参考等详细信息。

| 项目属性 | 值 |
|---------|---|
| 框架 | React 18 + TypeScript |
| 构建工具 | Vite 6 |
| 样式方案 | SCSS Modules + CSS Custom Properties |
| 包管理 | pnpm |
| 部署方式 | GitHub Actions → tar/SCP → Nginx |
| 线上地址 | coins.family-chen.com |

## 数据流

### 构建时数据管线

```
docs/target/*.md (26 个 Markdown 文件)
        │
        ▼  parse-coins-data.mjs
data/coins.json (1.4 MB 完整数据)
        │
        ▼  split-coins-data.mjs
   ┌────┴────┐
   ▼         ▼
public/data/   public/data/
coins-summary   detail/
.json           0.json ~ 25.json
(228 KB)        (1.1 MB 共)
```

### 运行时数据加载

```
页面加载
  │
  ├─ useSummaryData()  ──fetch──▶ /data/coins-summary.json
  │   （preload 提示，带缓存去重）
  │
  ├─ 朝代切换
  │   └─ prefetchDynasty() ──fetch──▶ /data/detail/{index}.json
  │      （内存 Map 缓存 + 请求去重）
  │
  └─ 选择钱币
      └─ useCoinDetail() ──从缓存或 fetch──▶ 详情数据
```

### 图片加载策略

| 场景 | 图片类型 | 加载方式 |
|------|---------|---------|
| 详情主图 | 原图 WebP/JPG | `<picture>` 元素 + `loading="lazy"` |
| 变体缩略图 | 150×150 thumb.webp | `<picture>` 元素 + `loading="lazy"` |
| 缩放查看 | 原图 WebP/JPG | 点击时加载 |

## 类型系统

```typescript
// 稀有度等级
type RarityLevel = '1' | '2' | '3' | '4' | '5' | '6' | 'default';

// 图片
interface CoinImage { src: string; alt: string; label?: string }
interface CoinImages { main: string; variants: CoinImage[] }

// 摘要（首屏加载）
interface CoinSummary {
  name: string; historicalPeriod: string; ruler: string;
  coreFeatures: string; estimatedValue: string;
  rarity: string; thumbnail: string;
}

// 详情（按需加载）
interface CoinDetail {
  castingTime: string; material: string; dimensions: string;
  obverseFeatures: string; castingCraft: string; coreBackground: string;
  variantsTable: VariantTableRow[]; images: CoinImages;
}

// 钱币
interface Coin {
  id: string; name: string; dynasty: string; dynastyIndex: number;
  summary: CoinSummary; detail: CoinDetail | null;
}

// 朝代
interface DynastyData { dynasty: string; dynastyIndex: number; coins: Coin[] }
```

## 组件架构

```
App (状态中心)
├── DynastyTabs        朝代标签栏，横向滚动，自动居中激活标签
├── CoinList           钱币列表，显示名称/稀有度/核心特征
├── SearchBar          搜索栏，200ms 防抖，键盘导航，最多 20 条结果
└── CoinDetail ⚡      钱币详情面板（React.lazy 懒加载）
    ├── CoinImage      图片查看器，<picture> + WebP + 缩略图 + 缩放
    ├── DetailSection  详情字段展示（铸造时间/材质/尺寸等）
    └── VariantsSection 版别体系表格，rowspan 合并
```

### 状态管理

App 组件是唯一的状态中心，无外部状态库：

| 状态 | 类型 | 说明 |
|------|------|------|
| `allData` | `DynastyData[]` | 从 useSummaryData 异步获取 |
| `activeDynastyIndex` | `number` | 当前选中的朝代 |
| `selectedCoin` | `Coin \| null` | 当前选中的钱币 |
| `sidebarOpen` | `boolean` | 移动端侧边栏开关 |

通过 `useMemo` 稳定派生数据（`activeDynasty`），配合子组件的 `memo()` 避免不必要的重渲染。

## 性能优化体系

### 构建层

| 优化项 | 实现方式 | 效果 |
|--------|---------|------|
| 代码拆分 | React.lazy 懒加载 CoinDetail | 首屏 JS 减少 ~40-60 KB |
| 数据分离 | 摘要 JSON 从静态 import 改为 fetch | 首屏 JS 减少 228 KB |
| 压缩 | terser + drop_console | JS 体积减少 5-10% |
| 预压缩 | vite-plugin-compression 生成 .gz + .br | 传输体积减少 60-70% |
| 图片格式 | WebP 转换（Q80）+ 缩略图生成 | 图片体积减少 25-35% |

### 运行时

| 优化项 | 实现方式 | 位置 |
|--------|---------|------|
| 数据懒加载 | 详情 JSON 按朝代 fetch + 缓存去重 | useCoinDetail |
| 图片懒加载 | `loading="lazy"` + `<picture>` WebP 优先 | CoinImage |
| 缩略图 | 150×150 thumb.webp 替代原图缩略 | CoinImage |
| 搜索预热 | `requestIdleCallback` 空闲构建索引 | search.ts |
| 组件缓存 | 所有组件 `memo()` + `useMemo` 稳定引用 | 全组件 |
| 资源预加载 | `<link rel="preload">` 摘要 JSON + 首朝代详情 | index.html |

### 网络层（Nginx）

| 资源路径 | 缓存策略 | 说明 |
|---------|---------|------|
| `/assets/` | 1 年，immutable | Vite 带内容哈希的 JS/CSS |
| `/images/` | 30 天，public | 图片可能随版本更新 |
| `/data/` | 7 天，public | 数据可能随版本更新 |
| `/fonts/` | 1 年，immutable | 字体文件极少变化 |
| `/index.html` | no-cache | 确保用户获取最新资源引用 |
| 所有文本资源 | gzip_static + brotli_static | 优先发送预压缩文件 |

### 构建产物对比

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 首屏 JS（未压缩） | 183 KB | 18.1 KB |
| 首屏 JS（brotli） | ~55 KB | ~5.8 KB |
| 摘要 JSON 传输 | 内联 JS，brotli ~55 KB | 独立 fetch，brotli ~17 KB |
| 图片总量 | 69 MB JPG | ~45-50 MB WebP |
| 缩略图单张 | ~80 KB | ~3-5 KB |
